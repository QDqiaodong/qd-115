# 设备总使用时长接口功能说明

## 1. 功能概述

新增设备总使用时长聚合接口，支持按设备维度统计累计使用分钟数、最近使用日期和单次最长使用时长，并可按设备类型和位置进行筛选。

## 2. 接口信息

| 项 | 说明 |
| --- | --- |
| **接口路径** | `GET /api/usage/device-summary` |
| **接口描述** | 获取设备使用时长汇总统计 |
| **请求方式** | GET |
| **返回格式** | JSON |

## 3. 请求参数

| 参数名 | 类型 | 必填 | 说明 | 示例 |
| --- | --- | --- | --- | --- |
| deviceType | String | 否 | 设备类型枚举值 | `SPEAKER`, `PROJECTOR`, `PLAYER`, `AMPLIFIER` |
| location | String | 否 | 设备位置（支持模糊匹配标准化） | `客厅`, `卧室`, `书房` |

### 设备类型枚举说明

| 枚举值 | 中文标签 |
| --- | --- |
| SPEAKER | 音响 |
| PROJECTOR | 投影仪 |
| PLAYER | 播放器 |
| AMPLIFIER | 功放 |

### 位置标准化说明

位置参数支持模糊匹配，系统会自动标准化处理：
- 输入"客厅电视墙"、"客厅吊顶"、"客厅设备柜"等均会标准化为"客厅"
- 输入"卧室床头柜"会标准化为"卧室"
- 完整映射关系参见 [LocationUtils.java](file:///Users/Admin/Desktop/solo-0601/qd-0601/qd-115/backend/src/main/java/com/avledger/util/LocationUtils.java)

## 4. 响应字段说明

| 字段名 | 类型 | 说明 |
| --- | --- | --- |
| deviceId | Long | 设备ID |
| deviceName | String | 设备名称 |
| deviceType | String | 设备类型枚举值 |
| deviceTypeLabel | String | 设备类型中文标签 |
| location | String | 设备原始位置 |
| normalizedLocation | String | 标准化后的位置 |
| totalMinutes | Integer | 累计使用分钟数 |
| totalHours | Double | 累计使用小时数（保留1位小数） |
| lastUsedDate | String | 最近使用日期（格式：yyyy-MM-dd） |
| maxSingleDuration | Integer | 单次最长使用时长（分钟） |
| maxSingleDurationHours | Double | 单次最长使用时长（小时，保留1位小数） |

## 7. 业务逻辑说明

1. **数据聚合**：遍历所有设备，对每个设备的所有使用记录进行聚合计算
2. **设备类型过滤**：仅返回指定类型的设备数据，不传则返回所有类型
3. **位置过滤**：对输入位置进行标准化后，与设备位置的标准化结果进行匹配
4. **排序规则**：默认按累计使用分钟数降序排列
5. **空数据处理**：无使用记录的设备不返回在结果中

## 8. 注意事项

1. 接口返回的 `totalHours` 和 `maxSingleDurationHours` 为格式化后的小时数，保留1位小数
2. 位置参数支持模糊匹配，如传入"客厅"会匹配所有标准化位置为"客厅"的设备
3. 无使用记录的设备不会出现在返回结果中
4. 日期格式为 `yyyy-MM-dd` 字符串
